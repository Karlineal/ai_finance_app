## 1. 理财推荐功能 - 高风险 ⚠️

### 1.1 监管要求

在中国，提供投资建议需要严格遵守以下法规：

| 法规 | 要求 |
|-----|------|
| **《证券法》第160条** | 从事证券投资咨询业务需经国务院证券监督管理机构核准 |
| **《基金法》** | 基金销售需持牌，不得承诺收益 |
| **《关于规范金融机构资产管理业务的指导意见》** | 不得承诺保本保收益 |
| **《金融营销宣传行为管理办法》** | 不得虚假宣传、不得误导消费者 |

### 1.2 具体风险

#### A. 资质风险（最严重）
```
风险场景：
- AI推荐"买入某基金"
- 用户跟随建议投资亏损
- 用户起诉要求赔偿

法律后果：
- 非法经营证券投资咨询业务
- 行政处罚：罚款、责令停业
- 民事赔偿：赔偿用户损失
- 刑事责任：情节严重者可能涉及刑事责任
```

#### B. 虚假宣传风险
```
风险场景：
- AI声称"稳赚不赔"
- 展示"历史年化收益XX%"
- 暗示保本保收益

法律后果：
- 违反《广告法》第25条
- 罚款：广告费用3-5倍，无法计算则20-100万
- 信用惩戒：列入经营异常名录
```

#### C. 责任归属风险
```
问题：
- AI推荐出错，谁承担责任？
- 开发者？平台？AI模型提供商？

司法实践：
- 目前AI责任认定尚不明确
- 但提供工具的一方通常承担主要责任
```

#### ✅ 安全的做法（强烈推荐）

## 2. 发票/收据识别 - 中风险

### 2.1 隐私风险

发票包含敏感信息：
- 商家名称（商业行为）
- 消费金额（财务信息）
- 消费时间（行为轨迹）
- 商品明细（个人偏好）

### 2.2 合规要求

| 法规 | 要求 |
|-----|------|
| **《个人信息保护法》** | 处理敏感个人信息需单独同意 |
| **《数据安全法》** | 重要数据需加密存储、本地化处理 |
| **《网络安全法》** | 网络运营者需保护用户信息安全 |

### 2.3 风险场景

```
场景1：数据泄露
- OCR服务提供商泄露发票图片
- 用户敏感消费信息被第三方获取

场景2：数据滥用
- 分析用户消费习惯用于广告推送
- 未经授权将数据用于训练模型

场景3：跨境传输
- 使用境外OCR服务（如Google Vision）
- 数据出境未经过安全评估
```

### 2.4 合规措施

```typescript
// 隐私友好的OCR实现

interface OCROptions {
  // 数据处理方式
  processingMode: 'cloud' | 'local';
  
  // 数据保留策略
  retentionDays: number;
  
  // 是否脱敏
  maskSensitiveInfo: boolean;
}

class PrivacyPreservingOCR {
  async recognize(imageBase64: string, options: OCROptions) {
    // 1. 优先使用本地OCR（如PaddleOCR）
    if (options.processingMode === 'local') {
      return this.localOCR(imageBase64);
    }
    
    // 2. 云端OCR时的隐私保护
    // 2.1 图片压缩（降低分辨率）
    const compressedImage = await this.compressImage(imageBase64);
    
    // 2.2 元数据清理
    const cleanedImage = await this.stripMetadata(compressedImage);
    
    // 2.3 使用可信服务商（国内）
    const result = await this.callDomesticOCR(cleanedImage);
    
    // 2.4 及时删除云端数据
    await this.deleteCloudData();
    
    return result;
  }
  
  // 数据脱敏
  private maskSensitiveData(text: string): string {
    // 隐藏部分商家名称
    // 隐藏精确时间（只保留日期）
    // 隐藏商品明细（只保留类别）
    return maskedText;
  }
}
```

#### 推荐做法：

1. **本地OCR优先**
   - 使用 PaddleOCR 等开源方案
   - 数据不离开用户设备
   - 隐私风险最低

2. **云端OCR的合规措施**
   - 选择国内服务商（百度、腾讯、阿里）
   - 签署数据处理协议（DPA）
   - 关闭服务商的数据存储和改进功能

3. **用户授权**
   ```
   隐私政策中明确说明：
   - 我们会处理您的发票图片
   - 目的是自动提取交易信息
   - 数据处理方式（本地/云端）
   - 数据保留期限
   - 用户有权删除数据
   ```

## 3. 支付宝/微信数据导入 - 中风险

### 3.1 平台服务条款

#### 支付宝服务协议（节选）
```
用户不得：
- 使用非支付宝提供的接口/程序访问支付宝服务
- 未经授权获取、收集其他用户数据
- 使用自动化工具抓取支付宝数据
```

#### 微信用户协议（节选）
```
用户不得：
- 干扰或破坏微信服务
- 使用第三方工具访问微信数据
- 未经授权收集其他用户数据
```

### 3.2 风险分析

| 导入方式 | 风险等级 | 合规性 | 建议 |
|---------|---------|--------|------|
| **官方CSV导出** | 🟢 低 | ✅ 合规 | **推荐** |
| **模拟登录爬取** | 🔴 高 | ❌ 违规 | **禁止** |
| **第三方API** | 🟡 中 | ⚠️ 灰色 | 谨慎使用 |

### 3.3 推荐的合规方案

#### 方案：用户主动导出CSV

```
用户操作流程：
1. 用户登录支付宝/微信网页版
2. 手动导出账单CSV
3. 在AI记账助手中上传CSV
4. 助手解析并导入数据

合规性分析：
✅ 用户主动操作
✅ 不违反平台ToS
✅ 数据所有权清晰
```

## 4. 整体合规框架建议

### 4.1 必需的法律文件

#### 隐私政策（Privacy Policy）
```
必须包含：
1. 数据收集范围（交易数据、发票图片等）
2. 数据使用目的（记账、分析、AI处理）
3. 数据处理方式（本地/云端）
4. 数据保留期限
5. 用户权利（查看、删除、导出）
6. 第三方服务（AI服务提供商列表）
7. 联系方式
```

#### 用户协议（Terms of Service）
```
必须包含：
1. 服务范围说明
2. 用户责任（不得导入他人数据）
3. 免责声明（AI建议仅供参考）
4. 知识产权
5. 争议解决
```

#### 免责声明（Investment Disclaimer）
```
必须在理财相关页面显示：

"免责声明：
1. 本应用提供的基金信息仅供参考，不构成投资建议。
2. 基金过往业绩不代表未来表现，投资有风险，入市需谨慎。
3. 本应用不提供投资顾问服务，不构成任何投资决策依据。
4. 用户在做出投资决策前，应咨询持牌投资顾问。
5. 因使用本应用信息导致的投资损失，本应用不承担责任。"
```

## 6. 总结与建议

### 核心建议

1. **理财功能必须谨慎**
   - 仅提供信息展示，不做推荐
   - 强制显示免责声明
   - 考虑与持牌机构合作或移除

2. **发票识别优先本地**
   - 使用PaddleOCR等本地方案
   - 云端处理需脱敏
   - 明确告知用户数据处理方式

3. **数据导入使用官方导出**
   - 引导用户手动导出CSV
   - 严禁自动登录爬取
   - 用户协议中明确责任

4. **完善法律文件**
   - 制定详细的隐私政策
   - 用户协议明确权责
   - 各功能页面显示免责声明
