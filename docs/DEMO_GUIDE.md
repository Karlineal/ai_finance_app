# iCookie 演示路径指南

## 演示流程（约 3-5 分钟）

**路径：** 首页  
**展示点：**
- 深色主题下的卡片式布局
- 按日期分组的交易列表
- 收入/支出金额的颜色区分
- 底部导航的交互反馈

**操作：** 滚动列表，展示流畅度

---

### 2. 记账 - 核心功能
**路径：** 点击底部 "+" 按钮  
**展示点：**
- Emoji 分类图标网格

1. 输入金额（如 35）
2. 选择分类（餐饮）

## 演示技巧

### 开场白（30秒）
"iCookie 是一款面向年轻用户的智能记账应用。今天我会展示它的核心功能：从快速记账到 AI 识别，从预算管理到存钱计划。"

### 关键卖点强调
1. **设计一致性**："所有页面遵循统一的设计系统，深浅主题无缝切换"
2. **交互反馈**："每次点击都有触觉和视觉反馈，让操作更有确定感"
3. **功能完整**："不只是记账，还包括预算、存钱、订阅管理等完整财务工具"
4. **技术实现**："基于 Flutter 开发，一套代码同时支持 iOS 和 Android"

### 应急备案
- 如果 AI 识别功能不稳定：跳过实际拍照，仅展示界面
- 如果数据为空：提前准备几笔示例交易
- 如果动画卡顿：说明是开发模式，Release 版本更流畅

## 常见问题准备

**Q: 数据存储在哪里？**
A: 本地 SQLite 数据库，同时支持 Web 平台的内存存储。

**Q: 支持云同步吗？**
A: 当前版本是本地优先，未来会加入云同步功能。

**Q: AI 记账的准确率如何？**
A: 基于 Kimi API，对标准发票识别率较高，复杂场景会提示用户确认。

**Q: 为什么叫 iCookie？**
A: 灵感来自 Cookie 记账，i 代表 intelligent（智能）和 individual（个人）。
